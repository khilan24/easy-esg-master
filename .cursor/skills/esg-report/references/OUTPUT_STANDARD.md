# ESG 投研报告 — 输出与内容标准（周报 / 日报）

本文档定义报告 JSON 结构、Word 占位符、内容规范与精简版研究提示，供无项目代码的用户仅凭本 skill 产出同等交付标准的 **ESG 投研周报**（上周一至上周日）或 **ESG 投研日报**（昨日）。周报与日报共用同一 JSON 结构；仅标题与周期不同：周报为 `ESG投研周报`、多日区间，日报为 `ESG投研日报`、单日区间。

---

## 一、JSON 报告结构

报告必须输出为以下结构的 JSON，便于程序化填充 Word 或二次处理。

### 1.1 顶层结构

```json
{
  "report_metadata": { ... },
  "report_content": { ... }
}
```

### 1.2 report_metadata

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | string | 报告标题。周报：`ESG投研周报（YYYY年MM月DD日–YYYY年MM月DD日）`；日报：`ESG投研日报（YYYY年MM月DD日）` |
| `report_period` | object | 见下表 |
| `generation_time` | string | 生成时间，建议 ISO 8601 |
| `agent` | string | 可选，生成方标识 |
| `model` | string | 可选，模型标识 |

**report_period 结构：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `start_date` | string | 周期开始日期，如 `2026-01-19` |
| `end_date` | string | 周期结束日期，如 `2026-01-25` |
| `date_range` | string | 展示用日期范围，如 `2026年1月19日–2026年1月25日` |

### 1.3 report_content

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | string | 与 metadata.title 一致或简写 |
| `hotspot_focus` | string | 热点聚焦全文（多段纯文本，段间用 `\n` 分隔） |
| `environmental` | object | 环境（E）章节，见下 |
| `social` | object | 社会（S）章节，见下 |
| `governance` | object | 公司治理（G）章节，见下 |

**各章节对象结构（environmental / social / governance）：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `section_title` | string | 章节标题，如 `环境（E）动态`、`社会（S）动态`、`公司治理（G）动态` |
| `news_items` | array | 新闻条目列表，每项见下 |

**news_items 每项：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | string | 该条新闻的小标题 |
| `content` | string | 正文（含导语、核心细节、深度点评、结尾）；**末尾必须单独成行**写 `资料来源：<可访问的 URL>` |

### 1.4 示例（片段）

```json
{
  "report_metadata": {
    "title": "ESG投研周报（2026年1月19日–2026年1月25日）",
    "report_period": {
      "start_date": "2026-01-19",
      "end_date": "2026-01-25",
      "date_range": "2026年1月19日–2026年1月25日"
    },
    "generation_time": "2026-01-26T10:00:00+08:00"
  },
  "report_content": {
    "title": "ESG投研周报（2026年1月19日–2026年1月25日）",
    "hotspot_focus": "【环境】本周…\n【社会】本周…\n【公司治理】本周…",
    "environmental": {
      "section_title": "环境（E）动态",
      "news_items": [
        {
          "title": "欧盟委员会通过《可持续产品生态设计条例》",
          "content": "导语与核心细节…\n深度点评…\n结尾。\n资料来源：https://example.com/news/123"
        }
      ]
    },
    "social": { "section_title": "社会（S）动态", "news_items": [] },
    "governance": { "section_title": "公司治理（G）动态", "news_items": [] }
  }
}
```

## 二、Word 模板占位符

若使用 Word 模板填充，模板中应使用**双花括号**占位符，与下表一一对应。填充时按 key 替换；未使用的新闻占位符（如某章只有 3 条新闻则 4–8 为空）应从模板中移除或留空，不保留“空占位符”。

| 占位符 Key | 说明 | 取值来源 |
|------------|------|----------|
| `{{ 日期范围 }}` | 报告周期展示 | `report_metadata.report_period.date_range` |
| `{{ 热点聚焦 }}` | 热点聚焦全文 | `report_content.hotspot_focus` |
| `{{ 环境章节标题 }}` | E 章节标题 | `report_content.environmental.section_title` |
| `{{ 社会章节标题 }}` | S 章节标题 | `report_content.social.section_title` |
| `{{ 治理章节标题 }}` | G 章节标题 | `report_content.governance.section_title` |
| `{{ 环境新闻标题1 }}` … `{{ 环境新闻标题8 }}` | E 每条新闻标题 | `environmental.news_items[i].title` |
| `{{ 环境新闻内容1 }}` … `{{ 环境新闻内容8 }}` | E 每条新闻正文 | `environmental.news_items[i].content` |
| `{{ 社会新闻标题1 }}` … `{{ 社会新闻标题8 }}` | S 同上 | `social.news_items[i].title` |
| `{{ 社会新闻内容1 }}` … `{{ 社会新闻内容8 }}` | S 同上 | `social.news_items[i].content` |
| `{{ 治理新闻标题1 }}` … `{{ 治理新闻标题8 }}` | G 同上 | `governance.news_items[i].title` |
| `{{ 治理新闻内容1 }}` … `{{ 治理新闻内容8 }}` | G 同上 | `governance.news_items[i].content` |

说明：每章最多支持 8 条新闻；不足 8 条时只填充 1..N，并清理 N+1..8 的占位符。

## 三、内容规范（与项目一致）

### 3.1 时效性

- 所有新闻/资讯的**发布日期**必须落在当周（报告周期）内，含首尾日。
- 若某领域（E/S/G）当周无符合要求的资讯，须明确写明：  
  `经查询，该期间无符合要求的重大环境/社会/治理（E/S/G）领域相关资讯`，不得编造或使用超期资讯。

### 3.2 章节结构

- **无开篇摘要、无结尾总结**；章节主体即为一条条新闻，从第一条新闻直接开始。
- 每条新闻：**一个小标题 + 一段正文**；正文末尾**单独成行**写资料来源。

### 3.3 单条新闻结构（E/S/G 通用）

1. **小标题**：一句说清“谁”和“干什么”，如：欧盟委员会通过《可持续产品生态设计条例》。
2. **导语**：Who、When、What、How much，并概括核心目标或宏观意图。
3. **核心细节**：用“在…方面”“聚焦于…”等结构化呈现；列具体数字、目标年份、重点领域、关键机制；监管类可引用具体条款。
4. **深度点评**：定性定位、影响分析、短期/中长期趋势、机会点。
5. **结尾**：直接后续影响与可观测行动，而非泛泛总结。
6. **资料来源**：**必须**单独成行，格式为 `资料来源：<可访问的 URL>`；若为多条来源，可多行，每行 `资料来源：<URL>`。

段落间用**单个换行符 `\n`** 分隔；整篇内不出现连续多个 `\n`。

### 3.4 热点聚焦

- **定位**：周报开篇摘要，精炼概括当周 E、S、G 核心动向与逻辑。
- **结构**：三个自然段，分别对应 **E（环境）、S（社会）、G（公司治理）**。
- **内容**：每段 2–4 句话；概括 1–2 项核心政策/事件/趋势；可点明政策/事件间关联及对市场、产业的共同影响；避免简单罗列，体现本周在更长ESG脉络中的意义。
- **语言**：正式、精炼的投研书面语。

### 3.5 合并与校验（若分 E/S/G 分别生成再合并）

- 仅保留发布日期在**报告周期内**的资讯；剔除超期内容。
- 确保每条资讯归属正确章节（E / S / G）。
- 合并顺序：标题 → 热点聚焦 → 环境动态 → 社会动态 → 公司治理动态。

## 四、格式与解析约定（便于程序化）

- **换行**：全文统一使用**单行换行符 `\n`**；多行换行需规范为单个 `\n`。
- **资料来源**：  
  - 识别行：以 `资料来源`、`Source:`、`[资料来源]` 等开头，或行内包含可提取的 URL。  
  - 输出格式：`资料来源：<URL>`，每条来源单独一行；无 URL 时写 `资料来源：` 也可接受，但优先保留可访问链接。
- **标题与正文解析**：  
  - 一条新闻 = 一个小标题 + 一段正文（至下一小标题或章节结尾）。  
  - 正文末尾若已有“资料来源”行，解析时保留在 content 末尾；若资料来源在别处，应移动到该条新闻 content 的末尾。

---

## 五、研究深度要求（E/S/G 章节通用）

撰写 E、S、G 任一章节时，应达到以下研究深度（可与 Deep Research / 多轮检索配合）：

- 广泛搜索并验证**至少 10–20 个不同信息来源**。
- 对重要资讯做多维度分析（政策背景、行业影响、市场反应等）。
- 对关键信息做**交叉验证**。
- 达到**专业投研报告**的深度与广度，不过早结束研究。

## 六、精简版提示（可直接用于无项目环境）

以下为可替换占位符的提示要点，供仅凭本 skill 的用户调用大模型时使用。

### 6.1 日期占位符

- `{DATE_RANGE_ISO}`：如 `2026-01-19 至 2026-01-25`
- `{DATE_RANGE_CHINESE}`：如 `2026年1月19日–2026年1月25日`（含首尾）

### 6.2 章节研究（E/S/G）— 共性要点

- 身份：专业 ESG 研究员，撰写投研周报的「环境/社会/治理」章节。
- 研究深度：10–20 个来源、多维度分析、交叉验证、专业标准、充分迭代不早停。
- 时效性：仅限**上周**（上述日期范围内）的公开资讯；若无则明确声明“经查询，该期间无符合要求的重大环境/社会/治理（E/S/G）领域相关资讯”。
- 章节结构：无开篇摘要、无结尾总结，直接从第一条新闻开始。
- 新闻结构：每条**小标题 + 导语 + 核心细节 + 深度点评 + 结尾**；合理分段、避免分点；**每条末尾单独成行**写 `资料来源：原文链接`（链接须可访问）。

### 6.3 热点聚焦

- 输入：已完成的 E、S、G 三章内容。
- 任务：撰写周报开篇“热点聚焦”摘要。
- 结构：三个自然段，对应 E、S、G。
- 要求：每段 2–4 句；概括 1–2 项核心政策/事件/趋势；点明关联与对市场/产业的影响；避免简单罗列；正式、精炼书面语。

### 6.4 合并与成稿

- 输入：E、S、G 三章 + 热点聚焦。
- 校验：剔除报告周期外的资讯；核对 E/S/G 归属。
- 输出结构：标题（含日期范围）→ 热点聚焦 → 环境（E）动态 → 社会（S）动态 → 公司治理（G）动态。
- 若最终以 JSON 输出，须符合本文档第一节的 JSON 结构；若需填 Word，须符合第二节的占位符约定。

按本文档执行后，产出的 JSON 与 Word 报告可与本项目 `main.py` + 模板填充流程的交付标准一致，便于互换或统一归档。
